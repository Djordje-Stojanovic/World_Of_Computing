## Promotion Rationale

The Chapter 19 prose already distinguishes code-as-language from Chapter 18's general tools and Chapter 20's terminal-agent case study. These visuals make that boundary easier to read. They show code generation as a supervised workflow and benchmark results as harness-bound evidence, not as proof of productivity, autonomous engineering, security quality, or broad replacement.

## Remaining Blockers

- Exact HumanEval numbers, model sizes, code training-data composition, and benchmark comparisons.
- Copilot adoption, paid seats, productivity lift, code quality, and security quality.
- Live coding leaderboards or current best-coding-model claims.
- Production software-engineering transfer from contest benchmarks.
- Autonomous engineering, broad replacement, employment displacement, and secure-deployment claims.
