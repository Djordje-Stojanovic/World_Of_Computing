# I-0246 Chapter-Openers Beauty Package

Status: promoted planning package for pass I-0246. This pass does not rewrite the live chapter files. It defines the opener contract that the next prose pass should use before touching chapter text.

## Purpose

The book now has a front door, but each chapter also needs a smaller door: a title, a promise, a first beat, and a first visual that tells the reader why to keep going. The opener should create motion before apparatus arrives.

## Package

The executable board is `data/chapter_openers_package_i0246.tsv`. It contains 24 rows, one per canonical main chapter, with:

- official title and current file path,
- one-sentence chapter promise,
- opening-beat direction,
- candidate opening figure and asset,
- reader-effect job from the visual-addiction audit,
- claim guardrail,
- next integration gate.

## House Rule

Every chapter opener should do three things in this order:

1. Give the reader a concrete pressure or image.
2. Name the chapter's question in plain language.
3. Let the first figure clarify the pressure without pretending to prove more than its sources can support.

## Highest-Risk Openers

## Integration Notes

The next live prose pass should edit only the first visible screen and closing handoff of selected chapters, using this board as the control surface. It should not add unsupported scenes, invented interviews, unlicensed imagery, or broad product/economic claims.
