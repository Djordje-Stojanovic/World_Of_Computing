# Recurring Motif System - I-0224

Status: promoted visual/narrative system pass, 2026-05-26.

This pass adds a controlled motif layer on top of the existing visual grammar. The goal is not decoration. Each motif must teach the reader how to see a recurring mechanism:

| Motif | Job |
|---|---|
| Token | Make next-token prediction, tokenization, context accounting, and token economics visible. |
| Prompt | Mark the human or system instruction boundary. |
| Time | Carry chronology, release status, roadmap caveats, and the May 24, 2026 cutoff. |
| Chip | Tie model capability to accelerators, memory, interconnect, and software substrate. |
| Power | Keep electricity, interconnection, cooling, and physical supply visible beneath "AI factory" language. |
| Context | Show what is inside or outside the model's view. |
| Agency | Show bounded action through tools, permissions, tests, review, and handoff. |

The governing rule: a motif may recur only when it carries a claim-control job. If it does not clarify mechanism, chronology, source boundary, or blocked inference, it should not appear.
