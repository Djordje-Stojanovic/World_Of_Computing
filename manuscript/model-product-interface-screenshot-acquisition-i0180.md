# Model Product Interface Screenshot Acquisition - I-0180

Pass I-0180 turns major assistant, copilot, IDE, API, and agent surfaces into private-use screenshot slots. The goal is product texture: readers should see that the history moved from papers and APIs into ordinary work surfaces, but the screenshots must not become evidence for adoption, revenue, productivity, market share, safety, or autonomy.

## Capture Priorities

- P0 surfaces: ChatGPT launch, ChatGPT Plus, ChatGPT Enterprise, ChatGPT tools/GPTs, OpenAI API tools docs, OpenAI Codex, GitHub Copilot, Gemini/Bard, Claude, and Claude Code.
- P1 surfaces: Microsoft 365 Copilot and Cursor-style IDE surfaces, useful where Chapters 7 and 19 need workplace and IDE texture after the P0 assistant/coding-agent ladder is placed.
- Every capture needs private-use path, viewport, capture timestamp, URL, page title, raster sha256, source HTML sha256, and a rights/fair-use decision before publication.

## Caption Contract

The allowed caption form is narrow: source actor, source date or access date, interface/product surface, and chapter role. The blocked caption form is broad: no user counts, ARR, productivity gain, market share, model rank, security success, safety success, workflow automation success, or developer replacement unless a separate same-scope source supports that exact inference.

## Ledgers

- `data/model_product_interface_screenshot_acquisition_i0180.tsv` maps each desired screenshot to source IDs, target private-use paths, viewports, and blockers.
- `data/model_product_interface_capture_queue_i0180.tsv` is the executable capture queue for the future browser/hash pass.
- `data/model_product_interface_caption_blockers_i0180.tsv` defines the reusable caption firewall for product-interface visuals.

This pass deliberately commits no raster screenshots. It records only the acquisition plan, private-use slots, and provenance requirements.
