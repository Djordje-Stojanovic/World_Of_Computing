# Product Interface Exhibit Shortlist - Pass I-0193

This pass converts the I-0180 product/interface screenshot acquisition queue into a ranked exhibit shortlist. The aim is reader recognition without business-performance leakage: a product page can show that a lab or platform presented a surface to the world, but it cannot prove adoption, revenue, productivity, safety, market rank, autonomy, or current behavior.

## Recommended First Wave

The first capture wave should be eight surfaces: ChatGPT launch, ChatGPT Enterprise, Gemini/Bard, Claude, GitHub Copilot, Claude Code, OpenAI API tools, and OpenAI Codex. Together they cover the shift from chat to work surfaces, developer surfaces, coding assistance, and repository-agent packaging without making the book look like a brand catalog.

ChatGPT Plus, Microsoft 365 Copilot, GPTs/plugins, Cursor, and Google developer/API surfaces remain useful reserves. They should enter the final exhibit program only when they add a distinct chapter job: paid-product transition, office-app distribution, configurable assistants, AI-native IDEs, or developer-platform conversion.

## Deferrals

Model cards, system cards, pricing pages, and benchmark pages should not be treated as ordinary product screenshots. They are evidence surfaces with their own failure modes: quote limits, source dates, dynamic pricing, metric definitions, model versions, scaffolds, sample budgets, and current-page drift. They should flow through source-card, pricing, or benchmark-specific passes before publication.

## Caption Firewall

Every final screenshot caption must name the source owner, source date or capture date, and the exact surface role. The caption should not imply usage scale, subscriber count, revenue, ARR, productivity, code quality, deployment success, market share, model superiority, safety success, security effectiveness, developer replacement, or unsupervised autonomy. Those claims need same-scope evidence outside the screenshot.

## Outputs

- `data/product_interface_exhibit_shortlist_i0193.tsv`: 24 ranked candidates across product screenshots, source-card reserves, benchmark deferrals, pricing/model-card reserves, and one possible derived coding-agent comparison.
- `data/product_interface_exhibit_choose_drop_i0193.tsv`: 8 choose/drop decisions governing first-wave captures, reserves, deferrals, captions, and manifest timing.

No new screenshot, raster, source-card artwork, or manifest asset was created in this pass.
