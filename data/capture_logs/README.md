# Capture Logs

Pass-specific capture logs live here. Start from `data/capture_templates/capture_log_template.tsv`.

Committed logs should contain hashes, timestamps, source IDs, asset IDs, rights notes, and blockers. They should not contain heavyweight source media or long copyrighted excerpts.

Valid statuses include:

- `captured`
- `capture_blocked`
- `rejected`
- `superseded`
- `needs_rights_review`
- `needs_manifest_sync`
